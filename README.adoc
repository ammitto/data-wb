= Ammitto World Bank Sanctions Data Repository

== Purpose

This repository stores downloaded and processed sanctions data from the
World Bank Group's Debarment List (also known as the List of Ineligible
Firms and Individuals).

The data is maintained by the Ammitto project and used by the `ammitto` Ruby
gem for querying international sanctions information.

== Legal Basis

The World Bank debars entities and individuals under its procurement and
sanctions procedures:

* World Bank Procurement Regulations
* World Bank Sanctions Procedures
* Cross-Debarment Agreement with other Multilateral Development Banks (MDBs)

Debarment prevents sanctioned parties from participating in World Bank-financed
projects.

== Data Source

* **Authority**: World Bank Group - Office of Institutional Integrity
* **API Endpoint**: https://operations.worldbank.org/finance/wp-content/cache/developer/json/v2/all.json
* **Format**: JSON
* **Update Frequency**: As the World Bank updates the debarment list
* **Official Source**: https://www.worldbank.org/en/projects/operations/procurement/sanctions

== Directory Structure

```
.
├── downloaded/           # Raw JSON data from World Bank API
│   └── wb_sanctions_list.json
├── processed/            # Individual YAML files per sanction entity
│   └── {name}-{index}.yaml
├── reference-docs/       # Reference documentation
├── .github/workflows/    # GitHub Actions for automated updates
├── README.adoc           # This file
├── CLAUDE.md             # Guidance for Claude Code
└── update.log            # Timestamps of data updates
```

== Directory Descriptions

downloaded/:: Contains raw JSON data files as downloaded from the World Bank
API. The primary file is `wb_sanctions_list.json` which contains the complete
debarment list.

processed/:: Contains individual YAML files, one per sanction entity.
Files are named using the pattern `{primary-name}-{index}.yaml` where
index ensures uniqueness for entities with similar names.

reference-docs/:: Contains reference documentation about the World Bank
sanctions schema and data formats.

update.log:: Log file tracking when data was last updated with timestamps.

== Updating Data

Data can be updated using the ammitto gem:

[source,shell]
----
# From the ammitto gem directory
bundle exec ruby exe/ammitto fetch wb --format yaml --output-dir ./processed
----

Or via the adapters repository:

[source,shell]
----
# From the adapters/ directory
ruby adapter.rb
----

== Source Data Format

The processed YAML files in `processed/` follow this structure:

=== Person Entity
[source,yaml]
----
supp_id: 100451
supp_name: ZAKIR HOSSAIN
supp_type_code: I
entity_type: person
land1: BD
country_name: Bangladesh
supp_city: COMILLA
supp_addr: 371-KA JOBAIDA MANSION JHAWTALA - 3500
debar_type: O
debar_from_date: '2013-11-11'
debar_to_date: '2999-12-31'
debar_reason: 'Cross Debarment: ADB'
supp_elig_stat: X-DEBARRED
----

=== Organization Entity
[source,yaml]
----
supp_id: 200123
supp_name: EXAMPLE COMPANY LTD
supp_type_code: O
entity_type: organization
land1: CN
country_name: China
supp_city: Beijing
debar_type: T
debar_from_date: '2020-01-15'
debar_to_date: '2025-01-15'
debar_reason: 'Fraudulent Practice'
----

=== Field Definitions

[cols="1,4"]
|===
| Field | Description

| `supp_id`
| Unique supplier/entity ID

| `supp_name`
| Name of the sanctioned party

| `supp_type_code`
| Type: I (Individual) or O (Organization)

| `entity_type`
| Harmonized type: person or organization

| `land1`
| Country code (ISO 2-letter)

| `country_name`
| Full country name

| `supp_city`
| City of residence/registration

| `supp_addr`
| Full address

| `debar_type`
| Debarment type: O (Ongoing), T (Temporary)

| `debar_from_date`
| Debarment start date

| `debar_to_date`
| Debarment end date

| `debar_reason`
| Reason for debarment

| `supp_elig_stat`
| Eligibility status (X-DEBARRED, etc.)

|===

== Harmonized Data Format

After harmonization, entities are transformed into JSON-LD format:

=== Entity (JSON-LD)
[source,json]
----
{
  "@context": "https://www.ammitto.org/schema/v1/context.jsonld",
  "@id": "https://www.ammitto.org/entity/wb/100451",
  "@type": "PersonEntity",
  "names": [
    {
      "@type": "NameVariant",
      "fullName": "ZAKIR HOSSAIN",
      "script": "Latn",
      "isPrimary": true
    }
  ],
  "addresses": [
    {
      "@type": "Address",
      "city": "COMILLA",
      "country": "Bangladesh"
    }
  ],
  "sanctionEntries": [
    {
      "@id": "https://www.ammitto.org/entry/wb/100451",
      "@type": "SanctionEntry",
      "authority": {
        "@id": "https://www.ammitto.org/authority/wb",
        "name": "World Bank"
      },
      "regime": {
        "@type": "SanctionRegime",
        "code": "DEBARMENT",
        "name": "World Bank Debarment"
      },
      "effects": [
        {
          "@type": "SanctionEffect",
          "effectType": "debarment",
          "scope": "full"
        }
      ],
      "status": "active",
      "period": {
        "@type": "TemporalPeriod",
        "effectiveDate": "2013-11-11",
        "expiryDate": "2999-12-31"
      },
      "reasons": [
        {
          "@type": "SanctionReason",
          "description": "Cross Debarment: ADB"
        }
      ]
    }
  ]
}
----

=== Debarment Types

[cols="1,3"]
|===
| Code | Description

| `O`
| Ongoing debarment (indefinite)

| `T`
| Temporary debarment (fixed term)

| `S`
| Conditional debarment

|===

=== Debarment Reasons

* Fraudulent Practice
* Corrupt Practice
* Collusive Practice
* Coercive Practice
* Obstructive Practice
* Cross Debarment: ADB, AfDB, EBRD, IADB

== YAML Schema

Each processed YAML file follows the schema defined in
`ammitto/lib/ammitto/sources/wb/` using Lutaml::Model.

.Example YAML structure
[source,yaml]
----
names:
  - "PRIMARY NAME"
  - "ALIAS 1"
entity_type: person  # or "organization"
source: wb
country: Country Name
birthdate: "YYYY-MM-DD"
identifications:
  - type: "National ID"
    number: "XXX123456"
sanction_details:
  - project: "Project Name"
    start_date: "YYYY-MM-DD"
    end_date: "YYYY-MM-DD"
    grounds: "Grounds for debarment"
----

== Entity Types

The World Bank debarment list contains the following entity types:

Person:: Individual persons subject to debarment
Organization:: Legal entities, companies, and organizations

== Knowledge Graph Integration

This repository's processed YAML data feeds into the Ammitto knowledge graph.

When the harmonization process runs:

[source,shell]
----
# Auto-detect all data-* repos and harmonize
ammitto harmonize --scan --sources-dir /path/to/repos --output-dir ./api/v1
----

The wb data is transformed into:

* Individual JSON-LD nodes in `api/v1/node/entity/wb/{id}.jsonld`
* Sanction entries in `api/v1/node/entry/wb/{id}.jsonld`
* Lightweight search index entries
* Facet counts for filtering
