= Ammitto World Bank Sanctions Data Repository

== Purpose

This repository stores downloaded and processed sanctions data from the
World Bank Group's Debarment List (also known as the List of Ineligible
Firms and Individuals).

The data is maintained by the Ammitto project and used by the `ammitto` Ruby
gem for querying international sanctions information.

== Legal Basis

The World Bank debars entities and individuals under its procurement and
sanctions procedures:

* World Bank Procurement Regulations
* World Bank Sanctions Procedures
* Cross-Debarment Agreement with other Multilateral Development Banks (MDBs)

Debarment prevents sanctioned parties from participating in World Bank-financed
projects.

== Data Source

* **Authority**: World Bank Group - Office of Institutional Integrity
* **API Endpoint**: https://operations.worldbank.org/finance/wp-content/cache/developer/json/v2/all.json
* **Format**: JSON
* **Update Frequency**: As the World Bank updates the debarment list
* **Official Source**: https://www.worldbank.org/en/projects/operations/procurement/sanctions

== Directory Structure

```
.
├── downloaded/           # Raw JSON data from World Bank API
│   └── wb_sanctions_list.json
├── processed/            # Individual YAML files per sanction entity
│   └── {name}-{index}.yaml
├── reference-docs/       # Reference documentation
├── .github/workflows/    # GitHub Actions for automated updates
├── README.adoc           # This file
├── CLAUDE.md             # Guidance for Claude Code
└── update.log            # Timestamps of data updates
```

== Directory Descriptions

downloaded/:: Contains raw JSON data files as downloaded from the World Bank
API. The primary file is `wb_sanctions_list.json` which contains the complete
debarment list.

processed/:: Contains individual YAML files, one per sanction entity.
Files are named using the pattern `{primary-name}-{index}.yaml` where
index ensures uniqueness for entities with similar names.

reference-docs/:: Contains reference documentation about the World Bank
sanctions schema and data formats.

update.log:: Log file tracking when data was last updated with timestamps.

== Updating Data

Data can be updated using the ammitto gem:

[source,shell]
----
# From the ammitto gem directory
bundle exec ruby exe/ammitto fetch wb --format yaml --output-dir ./processed
----

Or via the adapters repository:

[source,shell]
----
# From the adapters/ directory
ruby adapter.rb
----

== YAML Schema

Each processed YAML file follows the schema defined in
`ammitto/lib/ammitto/sources/wb/` using Lutaml::Model.

.Example YAML structure
[source,yaml]
----
names:
  - "PRIMARY NAME"
  - "ALIAS 1"
entity_type: person  # or "organization"
source: wb
country: Country Name
birthdate: "YYYY-MM-DD"
identifications:
  - type: "National ID"
    number: "XXX123456"
sanction_details:
  - project: "Project Name"
    start_date: "YYYY-MM-DD"
    end_date: "YYYY-MM-DD"
    grounds: "Grounds for debarment"
----

== Entity Types

The World Bank debarment list contains the following entity types:

Person:: Individual persons subject to debarment
Organization:: Legal entities, companies, and organizations
